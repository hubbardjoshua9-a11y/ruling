# Reference — The Outsourcing Test (the core axis)

Every task is scored on one question at STAGE-3. This is the axis that decides most rulings. It is not "did the student *use* AI" — it is "could the student have outsourced their competency to AI and had it not show."

---

## The question
> **Could a student complete this task using AI, unsupervised, without the output reflecting their own competency or context?**

Run it literally. Imagine the best available general model, the task pasted in whole, no supervision, and ask what comes back. If what comes back would *pass* as the student's own evidence, the task is outsourceable.

## The four results
| Result | Meaning | Effect at STAGE-6 |
|---|---|---|
| **NONE** | AI cannot produce passable evidence (live performance, observed, contextualised to data AI can't see) | Axis passes |
| **LOW** | AI helps but the student's own context/judgment still carries the evidence | Axis passes |
| **MEDIUM** | AI produces most of it; specificity or coherence adds friction but doesn't close the gap | **REVISE** unless another axis compensates |
| **HIGH** | AI produces the whole thing, convincingly, with no friction | **REVISE** minimum; **REBUILD** if the method is structurally wrong for the evidence type |

## What pushes a task UP the scale (toward HIGH)
- Generic prompt with no context only the student holds ("explain the hierarchy of control").
- A scenario *provided to the student* — they complete a template from given facts (RULE-08). The facts are now in the prompt.
- "In your own words" with nothing context-specific to anchor it (RULE-10).
- "Describe a situation where you…" — fabricatable, unverifiable (RULE-11).
- Written/typed roleplay responses (RULE-12) — text is exactly what AI produces.
- Any artifact-only take-home with no presence, no live response, no unseen prompt.

## What pulls a task DOWN the scale (toward NONE)
- Contextualised to the **student's own workplace data** AI never sees (specifics it can't invent).
- **Live, unscripted** response (oral, video with a responsive element, observation).
- **Process/iteration capture** the student must have actually produced.
- An **unseen prompt released at assessment time** that the student responds to live or async-on-the-spot.
- **Third-party/supervisor sign-off** on real workplace performance.

## Compensation — the one nuance that saves a MEDIUM
A MEDIUM task is **not automatically REVISE**. If a **verification layer is already attached** to it — an unseen async recording at submission, a live oral covering the same criteria, a supervisor sign-off — that layer *compensates* and can pull the task to COMPLIANT. Score the task **as the package presents it**, including any attached verification, not the artifact in isolation. A HIGH task is never fully compensated by a generic disclosure statement (a prohibition is not a mechanism — RULE-14); it needs a real responsive anchor.

## Two reminders
- **Open-book split (RULE-27).** "AI is just another resource" is *partly* right and you split on it: knowledge that was always lookupable is acceptable **with a disclosure requirement added**; performance evidence still has to be *demonstrated*, not described — outsourcing it fails regardless.
- The test is about **authenticity of the student's competency**, the one rule of evidence AI strains most. Validity, sufficiency, and currency are checked elsewhere; this axis is authenticity.
