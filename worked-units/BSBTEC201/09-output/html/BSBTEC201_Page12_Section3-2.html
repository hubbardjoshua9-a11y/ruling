3.2 – Checking and amending your work

Never assume the first version is right. Build in a check before you treat anything as finished:

  • Read it back against the source you entered from.
  • Use the application's own tools. Word processors have spellcheck and grammar check. Spreadsheets can flag errors and can use data validation to stop wrong entries. Use them, but do not rely on them blindly, because a spellchecker will happily accept "their" where you meant "there".
  • Fix what is wrong, then re-check the part you changed, because corrections sometimes introduce new mistakes.
Examples across Australian workplaces
    • A retail head office assistant proofreads a price list against the supplier's email before it goes out, catching a $19 that should have been $90.
    • A school office worker uses spellcheck on a newsletter, then still reads it once more because spellcheck did not notice "from" where "form" was meant.
    • An accounting firm junior turns on data validation in a spreadsheet so a date column will not accept text by mistake.
