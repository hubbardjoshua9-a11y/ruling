Module 4 – Finalise, name and store your work

The last few minutes of a task matter as much as the first. A file that is well checked, sensibly named and stored in the right place can be found and trusted weeks later. One that is saved to the desktop with a vague name is effectively lost. This module covers closing out cleanly.

In this module

  • 4.1 The final review and edit pass
  • 4.2 File naming conventions and why they matter
  • 4.3 Storing, backing up and exiting safely

Assessment Readiness – Module 4

Before moving on, you should be able to:

  • carry out a final review and edit before saving, and remove anything that should not be in the file
  • name a file using a clear, consistent, date-led and versioned convention
  • store a file in the correct location with the expected folder structure, handle personal information securely, and exit the application properly.

Useful resources – Module 4
