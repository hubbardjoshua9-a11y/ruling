4.2 – File naming conventions and why they matter

A good file name lets anyone find the right file later without opening it. The single most useful habit is to be consistent. Where you include a date, write it as YYYY-MM-DD: for example 2026-06-04. This is the only date order that automatically sorts files into correct chronological order in a folder, so the newest version sits where you expect it.

Beyond the date, keep the rest of the name consistent too:

  • include a short, clear description of the content (for example Trade-Customer-Price-List)
  • show the version where it matters (v1, v2, or FINAL) so the latest one is never in doubt
  • avoid characters some systems reject, such as / \ : * ? " < >.

A practical pattern is date, then description, then version:

2026-06-04_Trade-Customer-Price-List_v2.xlsx

Always follow your organisation's naming standard where it has one. Many workplaces, especially in government and records-heavy industries, have a set convention you are expected to use.

Examples across Australian workplaces
    • A government department applies a strict records file-naming standard so any file can be traced and retrieved years later.
    • An accounting firm names every client working file with the YYYY-MM-DD date and a version number, so the most recent file is never in question at tax time.
    • A retail head office keeps a tidy shared-drive folder structure with consistent file names, so staff across different stores can find the current price list.
