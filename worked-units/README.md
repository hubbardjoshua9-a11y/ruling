# Worked units — Ruling on real Learnbuilt assessments

Three real Learnbuilt units, chosen to span the **affected ↔ AI-thin** spectrum so both of Ruling's axes are exercised. Each folder holds the Warrant split Ruling consumed (`input-split.md`) and the rulings it produced (`ruling-package.md`). BSBTEC201 additionally holds a full **EXECUTE** run — a committed, reversible change across md, docx and the Canvas/LMS render.

| Unit | Level | Where it sits | What it demonstrates |
|---|---|---|---|
| **[BSBTEC201](BSBTEC201/ruling-package.md)** — Use business software applications | Cert II | **AI-affected** | The headline: currency axis catches an authentic-but-obsolete task → **REVISE (modernise)** → **EXECUTE** commits it end-to-end (md + docx + Canvas), tracked and reversible. |
| **[BSBWHS411](BSBWHS411/ruling-package.md)** — Implement & monitor WHS | Cert IV | **mixed** | Both axes firing in one package: modernise the INTEGRATE tasks, hold the line on the RESIST ones, and a policy REVISE the INTEGRATE tasks force. |
| **[CHCDIV001](CHCDIV001/ruling-package.md)** — Work with diverse people | Cert IV | **AI-thin** | Restraint: the currency axis correctly **stays silent** (RULE-33). The only fault is an authenticity/evidence-type one, caught by Axis A. |

> **A note on "real files."** The rulings are made against the genuine Learnbuilt production assessments (the true before-state). The EXECUTE commit is performed into a **copy** inside this folder — so the demo is self-contained and the live production set is untouched — while Ruling's EXECUTE mode is fully capable of running on the production files directly (the change pack's `before/` makes any run reversible; a rollback test confirmed byte-identical restoration).

**The end-to-end proof:** `BSBTEC201/_ruling-change-packs/A2-step3-modernise/` — the authorising ruling, the changelog, `before/` + `after/`, and human-readable diff cards for md, docx and LMS.
